# Запуск проекта

Пошаговая инструкция развёртывания с нуля.

Связанные разделы: [[Установка компонентов]] · [[Схема проекта]] · [[.NET CI-CD]]

## Обзор

```
terraform apply  →  gitea-network + Gitea + runner + registry + k3d + ArgoCD
Gitea UI         →  http://gitea:3000
ArgoCD UI        →  http://127.0.0.1:30090
```

Репозиторий .NET создаётся **вручную** из `templates/dotnet/` — см. [[.NET CI-CD]].

---

## Развернуть всё

```powershell
cd terraform
terraform init
terraform apply
```

Terraform создаёт:

- Docker-сеть `gitea-network`
- Gitea (`http://gitea:3000`)
- act_runner (авторегистрация)
- Docker registry `docker-registry:5000`
- k3d-кластер `local` (провайдер `SneakyBugs/k3d`)
- ArgoCD (Helm) + GitOps Application (провайдер `argoproj-labs/argocd`)

Подтвердите: `yes`.

### (Опционально) Настройки

```powershell
copy terraform.tfvars.example terraform.tfvars
```

```hcl
admin_password        = "надёжный_пароль"
admin_email           = "you@example.com"
argocd_admin_password = "admin"
gitea_root_url        = "http://gitea:3000"
```

### Проверка

```powershell
docker ps --filter name=gitea
docker ps --filter name=gitea-act-runner
docker ps --filter name=docker-registry
docker ps --filter name=k3d-local
terraform output
```

Gitea: `admin` / `admin` (или пароль из `terraform.tfvars`).

**Site Administration → Actions → Runners** — runner **Online**.

### ArgoCD

```powershell
terraform output argocd_ui_url
terraform output -raw argocd_admin_password
```

Логин: `admin`. Создайте репозиторий из `templates/dotnet/` — см. [[.NET CI-CD]].

### Kubeconfig (опционально)

```powershell
terraform output -raw kubeconfig | Out-File -Encoding utf8NoBOM kubeconfig.yaml
$env:KUBECONFIG = (Resolve-Path kubeconfig.yaml)
kubectl get nodes
```
