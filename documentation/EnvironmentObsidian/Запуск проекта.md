# Запуск проекта

Пошаговая инструкция развёртывания среды с нуля.

Связанные разделы: [[Установка компонентов]] · [[Схема проекта]]

## Обзор

```
terraform/   →  gitea-network + Gitea + act_runner + registry + k3d
Gitea UI     →  http://gitea:3000 из контейнера в gitea-network (или проброс порта 3000)
```

---

## Развернуть всё

Перед apply убедитесь, что установлены компоненты из [[Установка компонентов]] — в том числе **k3d** в PATH (`k3d version`). Репозиторий .NET создаётся вручную из `templates/dotnet/` — см. [[.NET CI-CD]].

```powershell
cd terraform
terraform init
terraform apply
```

Terraform создаст:

- Docker-сеть `gitea-network`
- Gitea (`http://gitea:3000` в сети)
- act_runner с автоматической регистрацией
- Docker registry `docker-registry:5000` (только в сети)
- k3d-кластер `local` (kubeconfig: `terraform/.generated/kubeconfig-local`)

Подтвердите: введите `yes`.

### (Опционально) Настройки

```powershell
copy terraform.tfvars.example terraform.tfvars
```

Пример `terraform.tfvars`:

```hcl
admin_password = "надёжный_пароль"
admin_email    = "you@example.com"
gitea_root_url = "http://gitea:3000"
```

Затем снова `terraform apply`.

### Проверка

```powershell
docker ps --filter name=gitea
docker ps --filter name=gitea-act-runner
docker ps --filter name=docker-registry
docker ps --filter name=k3d-local
terraform output
```

Войдите в Gitea как `admin` / `admin` (UI: контейнер в `gitea-network` или проброс `3000:3000`).

В Gitea: **Site Administration → Actions → Runners** — runner должен быть **Online**.

## Первичная настройка для разработки

### Сменить пароль администратора

**Settings → Account → Change Password**

### Создать репозиторий

```powershell
# из контейнера в gitea-network
git clone http://gitea:3000/admin/my-repo.git

# SSH (порт 2222 проброшен на хост)
git clone ssh://git@gitea:2222/admin/my-repo.git
```

### Пример workflow

`.gitea/workflows/ci.yml`:

```yaml
name: CI
on: [push]
jobs:
  build:
    runs-on: self-hosted
    steps:
      - run: echo "Hello from Gitea Actions"
```

## Остановка и удаление

```powershell
cd terraform
terraform destroy
```

> [!caution] Данные
> Тома `gitea-data` и `gitea-runner-data` сохраняются в Docker после `destroy`. Полная очистка: `docker volume rm gitea-data gitea-runner-data`.

## Полезные команды

| Действие | Команда |
|----------|---------|
| Логи Gitea | `docker logs -f gitea` |
| Логи runner | `docker logs -f gitea-act-runner` |
| Healthcheck Gitea | `docker inspect --format='{{.State.Health.Status}}' gitea` |
| Outputs | `cd terraform; terraform output` |
| Токен runner | `terraform output -raw runner_registration_token` |
| kubectl | `kubectl --kubeconfig .generated/kubeconfig-local get nodes` |
