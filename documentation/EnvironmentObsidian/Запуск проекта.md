## Способ 1. Корневой стек (рекомендуется)

Один `terraform apply` поднимает **сеть**, **Gitea** и **act_runner**. Токен runner передаётся между модулями автоматически.

### Обзор

```
terraform/   →  сеть + Gitea + act_runner
Браузер      →  первый вход, смена пароля, создание репозитория
```

### Развернуть всё

```powershell
cd terraform
terraform init
terraform apply
```

Terraform создаст:

- Docker-сеть `environment-network` (в `network.tf`)
- Том и контейнер Gitea (модуль `gitea/`)
- Том и контейнер act_runner (модуль `act_runner/`)
- Учётную запись администратора
- Регистрацию runner с автоматической передачей токена
- **.NET репозиторий** `dotnet-app` с workflow CI/CD (если `create_dotnet_repo = true`)

> Перед apply включите **Kubernetes в Docker Desktop** и убедитесь, что **Git** установлен. Подробнее: [[.NET CI-CD]]

Подтвердите: введите `yes`.

### (Опционально) Настройки

```powershell
copy terraform.tfvars.example terraform.tfvars
```

Пример `terraform.tfvars`:

```hcl
network_name   = "environment-network"
admin_password = "надёжный_пароль"
admin_email    = "you@example.com"
```

Затем снова `terraform apply`.

### Проверка

```powershell
docker ps --filter name=gitea
docker ps --filter name=gitea-act-runner
terraform output
```

Откройте **http://localhost:3000**, войдите как `admin` / `admin` (или ваш пароль из `tfvars`).

В Gitea: **Site Administration → Actions → Runners** — runner должен быть **Online**.

## Способ 2. Раздельные модули (продвинутый)

Используйте, если модули нужно применять по отдельности. **Сеть всегда создаётся только в корневом стеке** (`terraform/network.tf`).

### Шаг 0. Создать сеть

```powershell
cd terraform
terraform init
terraform apply -target=docker_network.gitea
```

### Шаг 1. Gitea

Создайте `terraform/gitea/terraform.tfvars`:

```hcl
network_name = "environment-network"
```

```powershell
cd terraform\gitea
terraform init
terraform apply
```

Сохраните токен:

```powershell
terraform output -raw runner_registration_token
```

### Шаг 2. act_runner

```powershell
cd ..\act_runner
copy terraform.tfvars.example terraform.tfvars
```

В `terraform.tfvars` укажите сеть и токен:

```hcl
network_name              = "environment-network"
runner_registration_token = "ТОКЕН_ИЗ_ШАГА_1"
```

```powershell
terraform init
terraform apply
```

## Первичная настройка для разработки

### Сменить пароль администратора

**Settings → Account → Change Password**

### Создать репозиторий

```powershell
# HTTPS
git clone http://localhost:3000/admin/my-repo.git

# SSH (порт 2222)
git clone ssh://git@localhost:2222/admin/my-repo.git
```

### Пример workflow

`.gitea/workflows/ci.yml`:

```yaml
name: CI
on: [push]
jobs:
  build:
    runs-on: self-hosted
    steps:
      - uses: actions/checkout@v4
      - run: echo "Hello from Gitea Actions"
```

## Остановка и удаление

### Корневой стек

```powershell
cd terraform
terraform destroy
```

### Раздельные модули

Сначала runner, затем Gitea:

```powershell
cd terraform\act_runner
terraform destroy

cd ..\gitea
terraform destroy
```

> [!caution] Данные
> Том `gitea-data` сохраняется в Docker после `destroy`. Полная очистка: `docker volume rm gitea-data gitea-runner-data`.

## Полезные команды

| Действие | Команда |
|----------|---------|
| Логи Gitea | `docker logs -f gitea` |
| Логи runner | `docker logs -f gitea-act-runner` |
| Healthcheck Gitea | `docker inspect --format='{{.State.Health.Status}}' gitea` |
| Outputs | `cd terraform; terraform output` |
| Токен runner | `terraform output -raw runner_registration_token` |

Подробнее — в [[Справочник и устранение неполадок]].
