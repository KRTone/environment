# Установка компонентов

Перед запуском проекта на рабочей машине должны быть установлены и настроены перечисленные ниже компоненты. Инструкция ориентирована на **Windows 10/11**; на Linux/macOS шаги аналогичны, отличаются только установщики.

Связанные разделы: [[Главная]] · [[Запуск проекта]] · [[Схема проекта]]

---

## 1. Docker

**Зачем:** Gitea и act_runner работают как Docker-контейнеры. Terraform управляет ими через Docker API.

### Установка (Windows)

1. Скачайте [Docker Desktop](https://www.docker.com/products/docker-desktop/).
2. Установите с включённым бэкендом **WSL 2** (рекомендуется).
3. Перезагрузите систему при необходимости.
4. Запустите Docker Desktop и дождитесь статуса **Running**.

### Проверка

```powershell
docker version
docker ps
```

Обе команды должны выполниться без ошибок.

> [!important] Docker socket
> Runner монтирует Docker socket хоста (`/var/run/docker.sock` внутри Linux-контейнера), чтобы job'ы Actions могли запускать вложенные контейнеры. Docker Desktop на Windows предоставляет этот socket автоматически — отдельная настройка не нужна.

## 2. Terraform

**Зачем:** описание и развёртывание инфраструктуры (сеть, тома, контейнеры, токен runner).

**Требуемая версия:** `>= 1.5.0` (в проекте использовалась 1.15.x).

### Установка (Windows)

**Вариант A — winget:**

```powershell
winget install HashiCorp.Terraform
```

**Вариант B — вручную:**

1. Скачайте бинарник с [releases.hashicorp.com/terraform](https://releases.hashicorp.com/terraform/).
2. Распакуйте `terraform.exe` в каталог из переменной `PATH`.

### Проверка

```powershell
terraform version
```

## 3. PowerShell

**Зачем:** стек `gitea` после старта контейнера создаёт учётную запись администратора через `local-exec` с интерпретатором PowerShell.

### Требования

- **Windows PowerShell 5.1** или **PowerShell 7+** — оба подходят.
- Команда `powershell` или `pwsh` должна быть доступна из терминала, в котором вы запускаете `terraform apply`.

### Проверка

```powershell
$PSVersionTable.PSVersion
```

## 6. k3d

**Зачем:** модуль `terraform/k8s_cluster/` создаёт Kubernetes-кластер через [k3d](https://k3d.io/) в Docker-сети `gitea-network`. Без `k3d` в PATH команда `terraform apply` завершится ошибкой:

> k3d не найден в PATH

По умолчанию `create_k8s_cluster = true`. Отключить кластер: `create_k8s_cluster = false` в `terraform.tfvars`.

### Установка (Windows)

**Вариант A — winget:**

```powershell
winget install k3d.k3d
```

**Вариант B — Chocolatey:**

```powershell
choco install k3d
```

**Вариант C — вручную:** [k3d.io/stable/#installation](https://k3d.io/stable/#installation)

После установки **перезапустите терминал** (или IDE), чтобы обновился `PATH`.

### Проверка

```powershell
k3d version
```

Команда должна вывести версию без ошибок — из **того же** терминала, где запускаете `terraform apply`.

> [!important] PATH в IDE
> Если `k3d version` работает в PowerShell, но `terraform apply` в Cursor/VS Code падает с «k3d не найден», перезапустите IDE или запускайте Terraform из терминала, где `k3d` доступен.

## 7. Сетевые порты

Убедитесь, что порты свободны до запуска:

| Порт | Назначение |
|------|------------|
| `3000` | HTTP — веб-интерфейс Gitea |
| `2222` | SSH — `git clone` по SSH |

Проверка (PowerShell):

```powershell
netstat -ano | findstr ":3000"
netstat -ano | findstr ":2222"
```

Пустой вывод — порты свободны.

- [ ] Docker Desktop запущен (`docker ps` работает)

- [ ] **k3d** установлен и в PATH (`k3d version`) — если `create_k8s_cluster = true`

Когда все пункты отмечены → переходите к [[Запуск проекта]].
