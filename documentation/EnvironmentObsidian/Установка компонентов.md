# Установка компонентов

Перед запуском на рабочей машине нужны только **Docker** и **Terraform**. Дополнительное ПО (k3d CLI, kubectl, PowerShell-скрипты) **не требуется** — кластер и контейнеры создаются провайдерами Terraform.

Связанные разделы: [[Главная]] · [[Запуск проекта]] · [[Схема проекта]]

---

## 1. Docker

**Зачем:** Gitea, registry, act_runner и k3d работают как Docker-контейнеры. Terraform управляет ими через провайдер `kreuzwerker/docker`.

### Установка (Windows)

1. Скачайте [Docker Desktop](https://www.docker.com/products/docker-desktop/).
2. Установите с бэкендом **WSL 2** (рекомендуется).
3. Запустите Docker Desktop и дождитесь статуса **Running**.

### Проверка

```powershell
docker version
docker ps
```

> [!important] Docker socket
> Runner монтирует Docker socket хоста, чтобы job'ы Actions запускали вложенные контейнеры. Docker Desktop на Windows предоставляет socket автоматически.

## 2. Terraform

**Зачем:** описание и развёртывание всей инфраструктуры одной командой `terraform apply`.

**Требуемая версия:** `>= 1.5.0`

### Установка (Windows)

```powershell
winget install HashiCorp.Terraform
```

### Проверка

```powershell
terraform version
```

## 3. Git (клиент, опционально)

**Зачем:** клонирование шаблона `templates/dotnet/` и push в Gitea после развёртывания.

```powershell
winget install Git.Git
git --version
```

## 5. Сетевые порты

| Порт | Назначение |
|------|------------|
| `3000` | HTTP — Gitea |
| `2222` | SSH — Git |
| `6443` | Kubernetes API (k3d) |
| `30500` | Docker registry (push из CI) |
| `30090` | ArgoCD UI (NodePort) |

```powershell
netstat -ano | findstr ":3000"
netstat -ano | findstr ":2222"
```
