## Типичные проблемы

### Docker не запущен

**Симптом:** `error during connect` / `Cannot connect to the Docker daemon`

**Решение:** запустите Docker Desktop, дождитесь статуса Running, повторите `terraform apply`.

---

### Порт 3000 или 2222 занят

**Симптом:** `port is already allocated`

**Решение:** освободите порт или измените в `terraform/gitea/terraform.tfvars`:

```hcl
gitea_port     = 3001
gitea_ssh_port = 2223
gitea_root_url = "http://localhost:3001"
```

Затем `terraform apply`.

---

### Gitea container is not healthy

**Симптом:** ошибка provisioner `gitea_admin` при `terraform apply`

**Решение:**

1. `docker logs gitea` — проверьте ошибки старта
2. `docker inspect --format='{{.State.Health.Status}}' gitea`
3. Убедитесь, что порт 3000 доступен внутри контейнера
4. При повторном apply provisioner пропустит создание, если пользователь уже существует

---

### Runner offline в Gitea

**Симптом:** в **Actions → Runners** runner не появляется или Offline

**Проверки:**

1. Токен в `act_runner/terraform.tfvars` совпадает с `terraform output -raw runner_registration_token` из стека gitea
2. `docker logs gitea-act-runner`
3. Сеть `environment-network` должна существовать (`docker network ls`) — создаётся в `terraform/network.tf`
4. Gitea доступен: `docker exec gitea-act-runner wget -q -O- http://gitea:3000/api/healthz` (если runner в той же сети)

---

### kubectl: error reading [/k8s/] — recognized file extensions

**Симптом:** `error reading [/k8s/]: recognized file extensions are [.json .yaml .yml]` при deploy

**Причина:** nested `docker run -v $GITHUB_WORKSPACE/k8s:/k8s` монтирует путь с **хоста**, а не из job-контейнера — каталог `/k8s` в kubectl-контейнере пуст.

**Решение:** манифесты передаются в `kubectl apply -f -` через stdin; kubeconfig монтируется с хоста (`KUBECONFIG_HOST`) в nested `docker run`.

---

### kubectl: localhost:8080 connection refused

**Симптом:** `Get "http://localhost:8080/openapi/v2"` при deploy

**Причина:** kubectl не нашёл kubeconfig (`/kube/config` не смонтирован в job-контейнер) и использует дефолтный `localhost:8080`.

**Решение:** deploy использует `docker run` с `-v KUBECONFIG_HOST:/root/.kube/config` (путь хоста Windows) и подачу YAML через stdin.

---

### Duplicate mount point: /var/run/docker.sock

**Симптом:** `failed to create container: Duplicate mount point: /var/run/docker.sock` на шаге **Set up job**

**Причина:** в `act_runner/config.yaml.tpl` docker.sock был указан и в `options: -v ...`, и в `valid_volumes` — act_runner монтировал socket дважды.

**Решение:** оставить только `valid_volumes`. Выполните `terraform apply` и `docker restart gitea-act-runner`.

---

### Cannot find: node in PATH (actions/checkout)

**Симптом:** `Cannot find: node in PATH` на шаге Checkout

**Причина:** `actions/checkout` и `actions/setup-dotnet` — JavaScript-actions, им нужен Node.js в runner. act_runner его не предоставляет по умолчанию.

**Решение:** workflow использует `git clone` через shell и образы `mcr.microsoft.com/dotnet/sdk` / `docker:24-cli` вместо JS-actions. Выполните `terraform apply` для обновления репозитория, затем перезапустите workflow.

---

### actions/checkout: connection refused к localhost:3000

**Симптом:** `Unable to clone http://localhost:3000/actions/checkout` / `dial tcp [::1]:3000: connect: connection refused`

**Причина:** job-контейнер runner обращается к `localhost`, но Gitea на хосте недоступен из контейнера.

**Решение:** в Terraform задано:
- `GITEA__actions__DEFAULT_ACTIONS_URL=http://gitea:3000` (внутренний URL)
- job-контейнеры подключаются к `environment-network` (`act_runner/config.yaml.tpl`)

Выполните `terraform apply` для пересоздания Gitea и act_runner, затем перезапустите workflow.

Альтернатива (нужен интернет): `gitea_actions_url = "github"` в `terraform.tfvars`.

---

### Job зависает или не стартует

**Симптом:** workflow в очереди, нет логов

**Проверки:**

1. В `.gitea/workflows/*.yml` указано `runs-on: self-hosted` (или метка из `runner_labels`)
2. Actions включены в настройках репозитория
3. Runner Online
4. Docker socket доступен runner'у (`docker logs gitea-act-runner`)

---

### Ошибка сети при apply act_runner

**Симптом:** `network environment-network not found`

**Решение:** используйте корневой стек `terraform/` (сеть создаётся в `network.tf`) или сначала примените модуль `terraform/gitea`.

## Обновление образов

```powershell
cd terraform\gitea
terraform apply   # пересоздаст контейнер при смене тега образа

cd ..\act_runner
terraform apply
```

Или вручную:

```powershell
docker pull gitea/gitea:latest
docker pull gitea/act_runner:latest
```

Затем `terraform apply` в соответствующем стеке.

## Безопасность (минимум для dev-среды)

- Смените пароль `admin` после первого входа
- Не коммитьте `terraform.tfvars` с токенами и паролями
- Токен `runner_registration_token` даёт регистрацию runner — храните локально
- Среда по умолчанию рассчитана на **локальную разработку**, не на публичный интернет без дополнительной настройки (TLS, файрвол, сильные пароли)
