# Environment — локальная Git-платформа

Документация для разработчиков по развёртыванию **self-hosted** среды: Git-сервер **Gitea** + CI/CD **Gitea Actions** через **act_runner**. Инфраструктура описана в Terraform и поднимается в Docker.

## С чего начать

1. [[Установка компонентов]] — что установить на машину разработчика
2. [[Запуск проекта]] — пошаговый `terraform apply` для Gitea и runner
3. [[Схема проекта]] — архитектура, сети, порты и потоки данных

## Быстрые ссылки

| Ресурс                | Значение по умолчанию                 |
| --------------------- | ------------------------------------- |
| Веб-интерфейс Gitea   | http://localhost:3000                 |
| SSH для `git clone`   | `ssh://git@localhost:2222`            |
| Логин администратора  | `admin`                               |
| Пароль администратора | `admin` (смените после первого входа) |

## Структура репозитория

```
environment/
├── terraform/
│   ├── main.tf         # Корневой стек: сеть + Gitea + act_runner
│   ├── network.tf      # Общая Docker-сеть
│   ├── gitea/          # Модуль Gitea
│   └── act_runner/     # Модуль Gitea Actions runner
└── documentation/
    └── EnvironmentObsidian/   # Это хранилище
```

> Рекомендуемый способ запуска — один `terraform apply` из каталога `terraform/`. Отдельные модули в `gitea/` и `act_runner/` остаются для продвинутых сценариев.
